クイックスタート

Kaoooaiは、プロジェクトごとの開発規約、設計書テンプレート、レビュー観点、テスト方針を参照しながら、AIエージェントが計画・生成・検証を進める設計です。

導入初期は、いきなり全社標準へ展開せず、代表機能や数モジュール規模のPoCから始めることを推奨します。

project: name: sample-payment-api scope: detailed-design / code-generation / test-generation rules: - coding-standard - design-template - review-checklist approval: human_review_required: true

プラットフォーム構成

Kaoooaiは、INPUT資料、企業ルール、AIエージェント群、対抗AIレビュー、実行ログ・監査ログを組み合わせて運用します。設計・実装・テストの成果物を分断せず、同一のGround Truthを参照することで整合性を保ちます。

主な構成要素

  • Instructions：エージェントの基本方針、禁止事項、確認観点
  • Skills：特定タスクや社内ツール連携の実行手順
  • MCP Tools：コード解析、テスト実行、外部ツール連携
  • メモリ：プロジェクト固有の短期・長期ナレッジ

カスタムInstructions

APIレスポンス形式、命名規則、例外処理、ログ出力、レビュー観点など、組織固有の判断を自然言語または構造化された設定として反映します。

review_policy: check: - design_consistency - missing_exception_handling - maintainability - security_risk require_report_before_execution: true

Skills拡張

Skillsは、特定の業務手順や社内ルールをAIエージェントが再利用できるようにする仕組みです。たとえば、社内テンプレートに沿った設計書生成、特定フレームワークのテスト作成、Gitブランチ運用ルールなどを定義できます。

提供形態

Kaoooaiは、クラウド型、Enterprise型、Local型の3つの提供形態を検討できます。導入スピード、データ管理、ネットワーク制約、監査要件に応じて構成を選び、PoC時に技術条件と運用条件を確認します。

Local AI / オンプレミス

セキュリティ要件が厳しい組織では、社内サーバーやオンプレミス環境でのLocal AI構築を検討できます。外部送信制限、GPU環境、利用モデル、監査ログ、権限管理の要件を確認したうえで、構成を設計します。

実際の構成は、お客様のネットワークポリシー、対象コード、性能要件により個別確認が必要です。
